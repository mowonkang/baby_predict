# EduPath 앱 디자인 (시작 문서)

웹앱(현재 동작) → 모바일 앱으로 확장하기 위한 디자인·설계 출발점.

---

## 1. 현재 상태 (구현됨)
- **웹앱(모바일 우선)**: `backend_edu/static/index.html` — FastAPI가 서빙하는 단일 페이지 앱.
- **PWA 기초**: `manifest.webmanifest` + `icon.svg` + 테마 컬러 → 휴대폰에서 "홈 화면에 추가" 시 앱처럼 실행.
- **화면 흐름(3단계)**: 프로필(나이/예산) → 적성 진단 설문 → 결과 대시보드.

## 2. 디자인 시스템
| 항목 | 값 |
|------|-----|
| 브랜드 컬러 | `#6c5ce7`(보라) / 강조 `#00b894`(그린) / 정보 `#0984e3` / 경고 `#e17055` |
| 배경/카드 | `#f4f4fb` / `#ffffff`, 카드 radius 16, 그림자 약하게 |
| 타이포 | 시스템 한글 폰트(Apple SD Gothic Neo / Malgun Gothic), 본문 14px |
| 레이아웃 | 최대 폭 480px(모바일 우선), 상단 그라데이션 헤더 + 단계 인디케이터 |
| 컴포넌트 | 카드, 리커트 5점 pill, 타임라인(로드맵), 흥미 바, 칩(과목/계열) |

## 3. 화면 (Screens)
1. **온보딩/프로필**: 단계 카드(미취학~고등) 또는 직접 나이 입력, 예산(선택).
2. **적성 진단**: 18문항(흥미 RIASEC 12 + 학습성향 6), 응답 카운터, 부분응답 허용.
3. **결과 대시보드**:
   - 적성 요약(흥미 6유형 바 + 상위 칩) + 권장 학습 방식(`study_mode`)
   - 교육 로드맵 타임라인(단계별 focus·활동·의사결정 포인트)
   - 추천 커리큘럼(이유 포함)
   - 고교 과목 추천(공통/일반/진로/융합선택)

## 4. API 연동(이미 구현)
| 화면 | 호출 |
|------|------|
| 진단 | `GET /api/survey` |
| 결과 | `POST /api/recommend`, `POST /api/pathway`, `POST /api/subjects` |

## 5. 네이티브 앱 로드맵 (다음 단계)
- **단기(권장)**: 현재 웹앱을 **완전한 PWA**로 — PNG 아이콘(192/512), 서비스워커(오프라인 셸 캐시) 추가 → 설치형 앱 경험. 별도 스토어 심사 불필요.
- **중기**: **React Native(Expo)** 또는 **Flutter**로 네이티브 앱.
  - 백엔드(FastAPI)는 그대로 API 서버로 재사용 → 화면만 네이티브로 재구현.
  - 권장: Expo(React Native) — 웹앱과 컴포넌트/로직 공유 용이, iOS/Android 동시 배포.
  - 푸시 알림(학년 진급·의사결정 시점 리마인드), 로그인/프로필 저장 추가.
- **스토어 출시**: Apple App Store / Google Play 개발자 등록 → 빌드 → 심사.

## 6. 앱 만들 때 유의 (리서치 반영)
- **개인정보**: 만 14세 미만 **법정대리인 동의** 플로우를 회원/프로필 단계에 내장(개인정보보호법 §22조의2). 아동용 처리방침 별도, 프로파일링 신중.
- **표현**: '합격/성적 보장'·'1위' 등 금지(표시광고법). 적성검사는 "참고용" 고지.
- 근거: [`교육현황_리서치.md`](교육현황_리서치.md) §6.
