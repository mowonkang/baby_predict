# 관찰형 인지 성향 프로파일 + 성취 하위스킬 상세 (설계)

> 문제의식(사용자 피드백):
> 1. 입력이 정성적(잘함/보통/부족)뿐 → **"부족"이 또래 대비 무엇이** 부족한지 알 수 없다.
> 2. 부모가 아이 **성향을 몰라서** 못 쓰는 경우가 있다.
> 3. "예술 계열" **한 번 클릭이 결과를 과결정**한다.
> 코드: `backend_edu/app/cognitive.py`(인지 성향), `subskill.py`(하위스킬). 무과금(LLM 호출 없음).

## 1. 참고한 검사 — 웩슬러(K-WISC-V)

K-WISC-V는 아동 지능을 **5개 지표**로 다차원 측정:
언어이해(VCI)·시공간(VSI)·유동추론(FRI)·작업기억(WMI)·처리속도(PSI).
점수는 **평균 100·표준편차 15**의 정규분포로, 같은 나이 집단 내 **상대 위치(백분위)**로 해석
(예: 115=상위 약 16%, 85~115 사이에 약 68%).

**우리가 차용한 것 / 차용하지 않은 것**
- 차용: 5영역 다차원 구조, "평균 100·SD 15" 해석 프레이밍, 백분위 밴드.
- 비차용(금지): 실제 문항·규준·IQ 산출. 웩슬러는 유료·임상 검사이므로, 우리는 **관찰형 참고
  프로파일**만 만들고 **"임상 지능검사·진단이 아니며 전문가 검사를 권장"**을 명시한다.

출처: [K-WISC 소개(나무위키)](https://namu.wiki/w/%EC%9B%A9%EC%8A%AC%EB%9F%AC%20%EC%A7%80%EB%8A%A5%EA%B2%80%EC%82%AC) · [IQ 분류(위키백과)](https://ko.wikipedia.org/wiki/IQ_%EB%B6%84%EB%A5%98)

## 2. 관찰형 인지 성향 (`cognitive.py`) — 성향을 몰라도 되게

- **부모가 성향을 고르지 않는다.** 대신 **관찰 가능한 행동** 10문항(영역당 2개)을 **또래 대비**로 답한다:
  `또래보다 많이 부족(0) · 조금 부족(1) · 또래와 비슷(2) · 조금 우수(3) · 또래보다 우수(4)`.
  예) 언어이해: "새 낱말을 빨리 익히고 잘 표현한다" / 유동추론: "규칙·패턴을 스스로 찾아낸다".
- **응답 척도(측정형)**: 각 행동을 **또래 대비 5단계**(또래보다 많이 부족~우수)로 평가. **점수화**: 영역 응답 평균 비율 p(0~1) → 참고 지표 `index = 100 + (p−0.5)·60` (55~145로 클램프).
  중앙(자주 근처)=100, 최고=~130, 최저=~70. 밴드: 우수/평균상/또래평균/평균하/보완필요.
  백분위 = 정규분포 CDF(index, 100, 15).
- **상대 강·약점**만 강조(절대 IQ 아님). 미응답 영역은 중립 처리(콜드스타트 안전).

## 3. 과결정 방지 — 삼각측량

능력치(`stats.build_stats`)는 이제 **4가지 신호**를 합산한다:
관심활동(RIASEC) + 학습성향 + 사교육 경험 + **행동 관찰(인지 성향)** + 과목 성취.
→ 인지 영역이 8각 능력치에 기여(`COG_TO_STAT`): 언어이해→언어, 시공간→예술·창의·수리,
유동추론→수리·논리·탐구, 작업기억→수리·언어, 처리속도→소량.
**"예술 한 번 클릭"의 가중치가 상대적으로 낮아져** 여러 근거로 프로파일이 형성된다.

## 4. 성취 하위스킬 상세 + 또래 대비 (`subskill.py`) — "무엇이" 부족한지

- 과목을 하위 스킬로 분해: 수학=**연산·개념·응용**, 국어=읽기이해·어휘·쓰기표현,
  영어=듣기·읽기어휘·문법쓰기, 과학=개념이해·탐구실험, 사회=개념암기·자료해석.
- 각 스킬을 **또래 대비 5단계**(최상위 상위10% · 우수 상위30% · 평균 · 부족 하위30% · 매우부족 하위10%)로
  평가 → **또래 평균 대비 밴드 + 백분위** + *무엇이* 약한지 + *어떻게* 보완/심화(무료·저가 자원 연결).
  예) 수학·연산 "부족(하위30%)" → "~27%, 계산 실수가 잦음 → 매일 10분 연산 루틴".
- 5단계 척도는 `levels.py`로 중앙화하며, 구값(잘함/보통/부족)·진단(BKT) 백분위도 정규화해 받는다.
  미니 진단을 하면 **실제 또래 백분위**가 상대 밴드로 자동 매핑된다.

## 5. API

| 메서드 | 경로 | 설명 |
|---|---|---|
| GET | `/api/cognitive-items` | 관찰형 행동 문항 10개 + 빈도 척도 |
| POST | `/api/cognitive` | 행동 응답 → 5영역 인지 성향 프로파일(밴드·백분위) |
| GET | `/api/subskills` | 과목 하위 스킬 목록 |
| POST | `/api/achievement` | (확장) 하위스킬 응답 → 또래 대비 상세 포함 |

입력은 `StudentProfile`에 `behaviors: {item_id→0~4}`(또래 대비 5단계), `subskills: {"수학:연산"→"부족"}` 추가.
`/api/subskills?age=`는 나이에 맞는 과목만 반환(초등 이하는 하위스킬 카드 자동 숨김).

## 6. 안전·한계

- **진단 아님**: 규준 표준화 검사가 아니라 부모 관찰 기반 참고치. 화면·문서·응답에 모두 고지.
- 문항·밴드는 예시이며 전문가 검수 후 확대 예정. 정밀 평가는 전문기관 검사를 안내.
- 전부 규칙 기반(LLM 호출 0) → 무료 티어 한계비용 0 유지.
