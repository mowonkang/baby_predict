# 지역 허브 + 엄마 커뮤니티 연동 설계

> 학부모가 실제로 원하는 것: **① 우리 동네 진짜 학원**, **② 지도로 위치 확인**,
> **③ 엄마들(맘카페) 실제 후기**, **④ 앱 안에서 직접 의견 나누기.**
> 코드: `backend_edu/app/local.py`(허브·딥링크), `community.py`(인앱 게시판). 무과금(LLM 호출 없음).

---

## 1. 왜 필요한가 (해자 관점)

- 우리 추천 엔진(무료강의·능력치·테크트리)은 "무엇을/언제"를 알려주지만, 실행 단계에서
  학부모는 **"그래서 우리 동네 어디?"** 와 **"먼저 다녀본 엄마 얘기"** 를 원한다.
- 이 둘을 채우면 **체류·재방문**이 늘고, 사용자 작성 후기·글이 쌓이는 **커뮤니티 데이터 네트워크
  효과**가 생긴다 → 후발주자가 복제하기 힘든 해자(`성장성_및_예측모델.md` §3.7).

## 2. 지역 허브 (`local.py`)

세 가지를 지역·과목 하나로 묶어 제공한다.

| 구성 | 방식 | 키 필요? |
|---|---|---|
| **실제 학원 목록** | 네이버 지역검색 API(실시간) → 없으면 수집 스냅샷(`data/academies_seed.json`) | 실시간만 키 필요 |
| **지도 딥링크** | 카카오맵·네이버지도 `?q={지역} {과목}학원` | ❌ 불필요 |
| **맘카페/후기 딥링크** | 네이버 카페글 검색(`where=article`)·통합 후기 검색 | ❌ 불필요 |

- **딥링크는 API 키 없이 항상 동작**한다(런타임 안전). 학원명·주소는 카카오맵 검색 링크로 바로 연결.
- **실시간 학원 조회**는 `NAVER_CLIENT_ID`/`NAVER_CLIENT_SECRET` 환경변수가 있으면 네이버
  지역검색 Open API를 표준 라이브러리(`urllib`)로 호출(추가 의존성 없음), 실패/미설정 시 스냅샷 폴백.
- 스냅샷은 **공개 정보(학원명·주소)** 이며 본 앱의 추천/평가가 아니다(표시광고·명예훼손 리스크 회피).

**API**: `GET /api/local-hub?region=&subject=` → `academies[] · map_links[] · community_links[]`.

## 3. 인앱 커뮤니티 (`community.py`)

외부 맘카페 링크와 별개로 **앱 안에서 직접** 글·댓글·공감을 남긴다.

- **글**: 지역 + 주제(수학학원·유아영어·사고력·예체능·진로·자유)로 필터.
- **댓글(의견 나누기)** · **공감(좋아요)**.
- 서버 JSON 저장(MVP, `data/community.json`). 빈 게시판 방지용 예시 시드 2건(파일 없을 때 1회 기록).
- 운영 확장: 로그인·DB·**신고/차단·욕설필터·개인정보 마스킹**, 만14세 미만 보호자 동의(개인정보보호법
  제22조의2). 사용자 작성 콘텐츠는 상업/비방/개인정보 게시 금지(신고 대상)로 명시.

**API**
| 메서드 | 경로 | 설명 |
|---|---|---|
| GET | `/api/community?region=&topic=` | 지역·주제별 글 목록 |
| POST | `/api/community` | 글 작성 |
| POST | `/api/community/{id}/comment` | 댓글(의견) 작성 |
| POST | `/api/community/{id}/like` | 공감(+1) |

## 4. 프론트

결과 화면에 두 카드 추가:
- **🏫 우리 동네 학원·지도·맘카페** — 지역·과목 입력 → 실제 학원 카드(지도 링크)·지도 딥링크·맘카페 후기 딥링크.
- **💬 엄마들 커뮤니티** — 지역·주제 필터, 글 목록(댓글·공감), 새 글/댓글 작성.

지역은 학원 추천에서 입력한 `region`, 과목은 보완 필요 1순위 과목으로 자동 프리필.

## 5. 안전·정책 요약

- 딥링크는 공개 검색 결과로 연결(캐시/색인 없음). 학원 스냅샷은 공개 사실 정보만.
- 인앱 후기·글은 사용자 생성물 → 신고/모더레이션 전제. 효과·평판은 정직하게(과장 금지) 커뮤니케이션.
