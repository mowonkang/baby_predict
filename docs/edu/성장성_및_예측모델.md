# EduPath 성장성 평가 + 페르소나 기반 예측 모델 설계

> 작성 2026-06 · 웹 리서치(에듀테크 2026 트렌드 / 지식추적 BKT·IRT) 기반.
> 목적: ① 추가 개발 요소, ② 플랫폼 성장성 평가(SWOT), ③ **아이 특성·페르소나 기반 예측 모델** 설계(무과금 방향).

## 1. 추가 개발 요소 (2026 트렌드 매핑)
검증된 흐름(개인화·게임화·분산복습·학습분석·홀리스틱 동반)을 우리 무과금 기조로:

| 트렌드 | EduPath 적용 | 비용 |
|--------|--------------|------|
| **게임화**(뱃지·레벨·적응형 챌린지) | 주간 플랜 완료 체크 → 스트릭·뱃지, 단원 클리어 | 0(프론트) |
| **분산 복습**(망각곡선 자동 스케줄) | 배운 단원의 복습일 자동 배치(1·3·7·14일) | 0(규칙) |
| **미니 진단**(적응형 출제) | 단원 3~5문항 → 정오답으로 숙련 추정(IRT/BKT) | 0(자체 문항·연산) |
| **학습 분석**(시간·재방문·완료율→성과) | 플랜 이행률·정체 조기경보 | 0(자체 집계) |
| **마이크로러닝** | '오늘 할 1개' 추천(짧게 자주) | 0 |
| **부모 리포트** | 주간 성취·이행 요약(이메일/PDF) | 0~저 |
| **또래 코호트 비교**(익명) | "같은 학년·적성 또래 대비" 위치 | 데이터 축적 후 |
| **웰빙·동기** | 과몰입 방지·격려 메시지, 불안 마케팅 배제 | 0 |

> 핵심: AI '효과'(개인화 +60% 참여)는 **자체 규칙·통계 모델**로 상당 부분 달성 가능 → LLM은 유료 Pro에서만.

## 2. 성장성 평가 (SWOT)

### Strengths
- **유일한 통합 포지션**: 0세~진로 전 생애주기 + 적성 + 학년별 교육과정 + 무료/공공 우선.
- **무과금 AI**(규칙·BKT/IRT) → 무료 사용자 확대해도 한계비용 0 → 사용자 확보에 유리.
- **해석가능**(왜 이 추천인지) → 신뢰 = 리텐션. 교육과정 출처 표기.

### Weaknesses
- **콘텐츠·실데이터 부족**(현재 예시 데이터), 진단 문항 없음 → 예측 정밀도 한계.
- **실사용자·네트워크 없음**(콜드스타트), 학원 거래·인증 미완.
- 단일 개발·검증된 학습효과 없음.

### Opportunities
- 사교육 **29조 시장**·참여율 80%, 공공데이터(EBS·어디가·커리어넷) 무료 활용.
- 고교학점제·2028 개편으로 **'과목·진로 설계' 수요 급증**.
- 2026 트렌드 "학생 생애 동반 + 진로 + 웰빙 통합 생태계"와 정확히 일치.
- B2B(학원 입점·광고)·해외(유사 구조 수출) 확장.

### Threats
- 대형 플레이어(메가스터디·콴다·밀크T)·자본·콘텐츠 우위.
- **데이터 확보 장벽**(개인화의 연료) — 초기 사용자 없으면 개인화 약함.
- 규제(미성년 개인정보·표시광고), 콘텐츠 신뢰·정확성 책임.

### 종합 성장성
- **아이디어·포지션·BM 구조: 강(强)**. **실행 리스크: 콘텐츠·진단 데이터·실사용자 확보**.
- 성패의 관건 = **'진단 데이터를 모으는 루프'**(미니 진단·플랜 이행 로그)를 빨리 돌려, 개인화 예측의 연료를 확보하는 것. → 아래 예측 모델이 그 핵심.

## 3. 페르소나 기반 예측 모델 설계 (핵심)

> 목표: 아이의 특성을 **인풋 데이터로 정밀히 포착(페르소나)**하고, 그에 적응하는 **예측**(숙련도·최적 학습·정체위험)을 한다. **모두 자체 연산(무과금)**, LLM은 선택적 유료.

### 3.1 입력 데이터 (페르소나의 연료)
| 층위 | 데이터 | 비고 |
|------|--------|------|
| **정적** | 적성(RIASEC 6), 학습성향(자기주도·협동·분석), 학년/월령, 지역·환경, 형제, 부모 기대 | 이미 수집 |
| **준정적** | 과목별 성취수준(자가/진단), 흥미 과목, 진로 목표 | 일부 수집 |
| **동적(시간축)** | **단원별 정·오답**, 학습시간, 완료율, 복습 이력, 재방문, **플랜 이행률**, 성취 변화 | 신규 — 핵심 연료 |

### 3.2 페르소나 = Learner Profile Vector (해석가능)
`persona = { 흥미(RIASEC), 학습성향, 과목·단원별 숙련도(0~1), 학습 페이스, 지속성/동기, 적정 난이도 }`
- 여기에 **페르소나 라벨**(클러스터): 예) "탐구형·자기주도 학습자", "관리형·사회형".
- 정적 입력으로 **콜드스타트** → 동적 로그로 **점점 정밀화**.

### 3.3 예측 타깃
1. 단원/과목 **숙련 확률**(mastery) 2. **다음 문제 정답 확률·적정 난이도** 3. 학습 **페이스·완료 시점**
4. **정체·이탈 위험**(조기경보) 5. **효과적 다음 학습**(자원·단원·방식) 6. 진로 계열 적합도(적성 기반, 보유).

### 3.4 방법 — 데이터 성숙도별 단계 (전부 자체 연산)
| 단계 | 방법 | 무엇을 | 비용 |
|------|------|--------|------|
| **L0 (현재)** | 규칙·가중치 | 성취도→시간배분(planner) | 0 |
| **L1** | **BKT-lite**(베이지안 지식추적) | 단원 정오답 → 숙련 확률 업데이트(p_init/p_learn/p_slip/p_guess), 다음 정답 확률 | 0(산수) |
| **L2** | **IRT-lite** | 학생 능력 θ·문항 난이도 b → 적정 난이도 출제·또래 비교 | 0 |
| **L3** | **유사 페르소나 협업필터링** | "비슷한 아이가 효과 본 경로/자원" 추천 | 0(축적 후) |
| **L4** | **경량 ML**(로지스틱/GBDT, scikit-learn) | 정체·이탈·완료율 예측(배치 학습·추론) | 0(자체) |
| **L5** | 딥러닝 지식추적(DKT/SAKT) | 데이터 충분·필요 시 정밀 KT | 학습=배치, 추론 경량 |
| (선택) | **LLM** | 대화형 설명·튜터 | 유료 → **Pro 티어에서만**(비용=수익 연동) |

> BKT/IRT는 지능형 튜터·적응형 출제의 표준이며 **해석가능 + 경량**이라 무료 티어에 적합. 딥러닝은 데이터가 충분할 때만.

### 3.5 콜드스타트 · 프라이버시
- **콜드스타트**: 적성+학년 규칙으로 시작 → 단원 진단 몇 개로 빠르게 보정.
- **프라이버시**: 미성년 데이터 법정대리인 동의, **온디바이스/익명 집계 우선**, 최소 수집(개인정보보호법, [`교육현황_리서치.md`](교육현황_리서치.md) §6).

### 3.6 우리 코드 적용 로드맵
1. **이벤트 로그 스키마**(단원·정오답·시간·이행) — 동적 데이터 수집 시작.
2. `mastery.py` (**BKT-lite**) + `/api/mastery` — 단원별 숙련도·다음 정답 확률.
3. `/api/persona` — 적성+학습성향+숙련도 통합 **페르소나 벡터·라벨** 반환.
4. **planner가 숙련도를 입력으로** 사용(현재 자가체크 → 숙련 확률로 정밀화) + 분산복습 스케줄.
5. 데이터 축적 후 L3(협업필터링)·L4(이탈 예측).

## 3.7 후발주자 방어 — 해자(Moat)/헷지 요소
특허가 아니어도 따라잡기 어렵게 만드는 방어선(중요도순):

1. **데이터 네트워크 효과(가장 강력)** — 진단·정오답·플랜 이행 로그가 쌓일수록 **BKT/IRT 파라미터·페르소나 클러스터·"비슷한 아이가 효과 본 경로"(협업필터링)**가 정교해진다. 후발주자는 사용자·데이터가 없어 같은 정밀도를 못 낸다. → **데이터 루프를 빨리·독점적으로 돌리는 것이 1순위 해자**.
2. **종단(생애주기) 프로필 락인** — 0세~진로까지 **한 아이의 성장 이력**이 누적되면 전환비용이 매우 커진다(경쟁사는 한 구간만). 이력·페르소나는 이사 가기 어렵다.
3. **양면 네트워크(학원·리뷰)** — 학부모↑→학원 입점↑→리뷰·정보↑→학부모↑. 리뷰·평판 데이터는 복제 불가.
4. **콘텐츠·매핑 자산** — 교육과정×학년×단원×무료자원×과목 매핑, 진단 문항 은행, 커리어·과목 매핑을 지속 축적·검수 → 시간이 드는 진입장벽.
5. **신뢰·브랜드** — 무료·공공 우선·정직한 효과 커뮤니케이션으로 쌓은 신뢰는 광고비로 단기 복제 불가.
6. **알고리즘 노하우** — BKT/IRT 파라미터 튜닝·페르소나 라벨링·적응 규칙은 공개 이론이라도 **우리 데이터로 보정된 값**이 경쟁력(모델 자체보다 데이터×튜닝).
7. **파트너십·공공 연계** — 학교·교육청·공공데이터·도서관 제휴는 선점 시 방어선.
8. **전환비용·습관** — 주간 플랜·스트릭·이력·동기화가 일상 루틴이 되면 이탈이 어렵다.

> 요약: **"모델"은 공개 이론이라 베낄 수 있지만, "데이터(정오답·이행·종단 이력)+튜닝+신뢰+양면 네트워크"는 시간과 사용자 없이는 못 만든다.** 그래서 **데이터 루프를 남보다 먼저·빠르게 돌리는 것**이 핵심 헷지다. (지식재산으로는 상표·DB권·영업비밀(파라미터/문항)·필요 시 UX 특허를 병행.)

## 4. 한 줄 결론
포지션·BM은 강하다. **성패는 '진단·이행 데이터 루프'를 돌려 개인화의 연료를 모으는 것**. 그 엔진을 **무과금 BKT/IRT 페르소나 모델**로 만들면, 대형 자본과 다른 길(무료·해석가능·생애주기)로 차별화할 수 있다. **다음 구현 1순위 = L1 BKT-lite 숙련 추적 + 페르소나 벡터.**
