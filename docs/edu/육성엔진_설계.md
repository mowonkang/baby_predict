# EduPath 육성 엔진 설계 (능력치 스탯 + 사교육 테크트리)

> "프린세스 메이커"처럼 **각 아이의 입력 → 능력치·관심 스킬을 정의하고, 사교육 테크트리 위에서
> 최적 루트를 제안**하는 고도화 엔진. **전부 규칙 기반(LLM 호출 없음) → 무료 티어 한계비용 0.**
> 관련: `../edu/성장성_및_예측모델.md`(예측모델 로드맵), `backend_edu/app/stats.py`·`techtree.py`

---

## 1. 개념 — 왜 게임식 육성 모델인가

- 학부모·학생에게 **"우리 아이는 지금 어떤 상태이고, 다음에 뭘 하면 되나"**를 한 화면으로 보여줘야 한다.
  리커트 설문·장문 리포트는 이탈률이 높다(온보딩 벤치마킹: Duolingo·Headspace의 탭 기반 질문 플로우가 완주율 우위).
- 게임의 **능력치(스탯) + 테크트리** 은유는 (1) 직관적이고 (2) 성장을 **시각적 변화**로 체감시키며
  (3) 다음 행동을 **루트**로 자연스럽게 유도한다 → 리텐션·전환의 훅.
- 우리는 이 은유를 **실제 교육 데이터/제도**에 정초한다: 능력치는 흥미(RIASEC)·학습성향·사교육 경험·과목
  성취에서 계산하고, 테크트리 노드는 실제 사교육 종목(몬테소리·영어·태권도·미술·코딩…)과
  무료·저가 대안(EBS·똑똑수학탐험대·공공시설)으로 채운다.

### 프린세스 메이커 대비 매핑

| 프린세스 메이커 | EduPath |
|---|---|
| 스탯(문학·무술·매력…) | 8각 능력치(언어·수리논리·탐구·예술·신체·사회성·리더십·창의) |
| 스케줄링(학당·아르바이트) | 사교육 테크트리 노드(계열별 단계) |
| 엔딩 분기 | 진로 마인드맵·유망 커리어(기존 `careers`·`pathway`) |
| 능력치 성장치 | 성취도 이력·BKT 숙련(기존 추적) |

## 2. 능력치 스탯 엔진 (`stats.py`)

### 2.1 8개 축 (8각형 레이더)

`언어 · 수리·논리 · 탐구 · 예술 · 신체 · 사회성 · 리더십 · 창의` — 각 0~100, 기본치 **40**에서 시작.

### 2.2 입력 → 능력치 합산 규칙 (무과금, 결정론적)

가산 신호는 4가지이며 **입력이 있을 때만** 반영(콜드스타트는 평평한 40 유지 → 정직).

1. **흥미(RIASEC)** — 중립(0.5) 초과분만 반영해 "뚜렷한 관심"일수록 크게.
   - realistic→신체·수리논리, investigative→탐구·수리논리, artistic→예술·창의,
     social→사회성·언어, enterprising→리더십·사회성, conventional→수리논리·언어
2. **학습성향** — self_direction→리더십, collaboration→사회성, analytical→수리논리·탐구
3. **사교육/활동 경험** — 선택한 종목의 `stat_weights`(예: 태권도→신체, 피아노→예술·창의,
   사고력수학→수리논리, 몬테소리→창의·탐구) × 스케일
4. **과목 성취** — 수학→수리논리, 과학→탐구, 국어/영어→언어, 사회→사회성 (잘함 +14 / 보통 +5)

최종 5~100 클램프. 상위(>기본치) 2~3축을 **강점**, 55 미만 하위 1~2축을 **보강**으로 표기.
헤드라인은 상위 2축 라벨(예: "탐구·언어형"). **어떤 입력이 반영됐는지**(`source_signals`)를 함께 노출해 신뢰 확보.

### 2.3 종합 레벨 · 육성 타이틀 · 성장 추적

- **종합(overall)** = 8축 평균 → `Lv.NN` + 레벨 라벨(새싹/성장/우수/탁월).
- **타이틀** = 최상위 축 기반 캐릭터 정체성(예: 탐구→"꼬마 과학자", 언어→"이야기꾼 언어가",
  리더십→"타고난 리더"). 콜드스타트는 "탐색가". → 프린세스 메이커의 캐릭터 정체성 훅.
- **성장 추적(프론트)**: 방문마다 능력치를 localStorage에 스냅샷 → **지난 방문 대비 Δ(▲/▼)** 를
  축별·종합으로 표시. "키우면 자란다"는 성장 체감 = 리텐션 루프의 핵심.

> 규제·윤리: 본 능력치는 심리·의학 '진단'이 아니라 **추천용 참고 프로파일**. 효과는 대규모로 미검증이므로
> 과장 없이 커뮤니케이션(공정위 제재 사례·`교육현황_리서치.md` §6 준수).

## 3. 사교육 테크트리 엔진 (`techtree.py`)

### 3.1 트리 구조 — 6계열 × 4티어

| 계열 | 대표 능력치 | tier0 유아(5~7) | tier1 초등(8~12) | tier2 중등(13~15) | tier3 고등·진로(16~19) |
|---|---|---|---|---|---|
| 어학 | 언어 | 영어노출·파닉스 | 회화·리딩 | 영문법·독해 | 수능영어·공인시험 |
| 사고력·수리 | 수리논리 | 교구·오감(몬테소리) | 사고력수학·코딩입문 | 수학심화·경시/정올 | 고등수학·수리논술 |
| 탐구·과학 | 탐구 | 자연관찰·과학놀이 | 창의과학실험·로봇 | 과학심화·영재원 | 이과심화·탐구활동 |
| 예술 | 예술 | 음악·미술놀이 | 피아노·미술 레슨 | 전공 심화 레슨 | 예중·예고·입시 |
| 체육 | 신체 | 신체놀이 | 태권도·수영 | 구기·전문종목 | 체대입시·선수 |
| 인성·리더십·표현 | 사회성 | 협동 그룹놀이 | 독서·논술·스피치 | 토론·리더십·동아리 | 진로탐색·대외활동 |

- 각 노드: `requires`(선행 노드)로 스타크래프트식 **테크 의존성**, `cost_band`(무료/저/중/고),
  `free_alt`(무료·저가 대안: EBS·똑똑수학탐험대·국립과학관·공공체육시설 등).
- 노드·비용대는 **예시(계열 카탈로그)** 이며 특정 학원·브랜드가 아니다(표시광고·명예훼손 리스크 회피).

### 3.2 추천 루트 매칭 (규칙)

1. `build_stats()`로 능력치 계산 → **강점 축** 도출.
2. 강점 축 → 대표 계열 매핑(언어→어학, 수리논리/창의→사고력, 탐구→과학, 예술→예술,
   신체→체육, 사회성/리더십→인성). 콜드스타트는 인성·사고력 기본 추천.
3. 나이 → 현재 티어(`<8`=0, `<14`=1, `<17`=2, else 3).
4. **이미 경험한 사교육 반영**: 입력한 활동 → `_ACTIVITY_PROGRESS`로 트랙별 도달 티어를 구해
   그 이하 노드를 **완료(done)** 로 표시하고, 추천 루트의 **시작 티어를 `max(나이 티어, 도달 티어+1)`**
   로 올린다(이미 한 건 다시 추천하지 않고 **다음 단계로 전진**). → 스타크래프트식 "테크 올리기".
5. 추천 계열의 **시작 티어(지금 이어갈 것) + 다음 티어(목표)** 노드를 루트로 강조.
6. **예산 인식**: `budget_max`가 빠듯(월 20만 미만)하면 루트 노드에 **무료·저가 대안**(`free_alt`)을
   함께 안내하고 "예산 절약 모드"로 표기.

프론트는 이를 **8각 레이더(성장 Δ) + 종합 레벨·타이틀 + 가로 스크롤 테크트리(완료✅/추천 하이라이트)
+ 개인화 루트 요약**으로 렌더.

## 4. API

| 메서드 | 경로 | 설명 |
|---|---|---|
| GET | `/api/extracurriculars` | 사교육·활동 계열별 선택지(쉬운 탭 입력용) |
| POST | `/api/stats` | 입력 → 8각 능력치(레이더) |
| POST | `/api/techtree` | 입력 → 전체 테크트리 + 능력치 + 추천 루트 |

입력은 기존 `StudentProfile`에 `activities:list[str]`(선택한 사교육 옵션 id) 필드 하나만 추가.

## 5. 쉬운 입력 UX (온보딩 벤치마킹 반영)

- **탭 기반 선택**(리커트 X): 나이 그리드 → 성취도 3버튼(잘함/보통/부족) → 관심활동·사교육 칩.
- **점진적 공개**: 최소 입력으로도 결과가 나오고(콜드스타트 안전), 입력을 더할수록 능력치·루트가 뚜렷.
- **즉시 시각 피드백**: 선택 즉시 결과 화면에서 레이더·테크트리가 채워짐(성장 체감 훅).
- **미니 진단 연계**: 기존 BKT 미니 진단 결과가 성취도로 흘러 능력치에 자동 반영.

## 6. 무과금 원칙·확장 로드맵

- 현재(L0~L1): **규칙 + 가중치 + BKT/IRT**만으로 능력치·루트 산출 → LLM 호출 0, 한계비용 0.
- L2+: 사용자 종단 데이터가 쌓이면 **협업필터링**("우리 아이와 비슷한 아이들이 다음에 탄 루트"),
  루트 성공률 기반 재랭킹으로 정밀화(설계: `성장성_및_예측모델.md`).
- **해자**: 능력치·루트·성취 이력이 누적된 **종단 프로필**은 후발주자가 복제하기 어려운 데이터 자산.
- LLM은 유료 Pro 티어의 **자연어 코칭 코멘트**로만 선택적 사용(무료 티어 원가 보호).

> 모든 노드·수치·비용대는 **예시**이며, 실제 학원/브랜드·효과를 보장하지 않는다(정직한 커뮤니케이션 원칙).
