# 아기 성향 기반 맞춤 추천 커머스 플랫폼 기획안

> **프로젝트 코드명**: `baby_predict`
> **문서 버전**: v0.1 (2026-06)
> **문서 종류**: 종합 사업·제품 기획안

---

## 0. 한 줄 정의 (Elevator Pitch)

> **"우리 아기 성향만 넣으면, 지금 사야 할 국민템과 발달 맞춤 용품·놀이를 알아서 추천해 주는 양육 큐레이션 플랫폼."**

초보 양육자가 "무엇을, 언제, 왜 사야 하는지" 고민하지 않도록, 아기의 **성향(기질) · 월령 · 거주지 · 가정 환경**을 입력하면 예측 모델이 발달 단계에 맞는 용품·놀이·교구를 추천한다. 수익은 **브랜드(업체)의 입점료/구독료 + 광고**로 만든다.

---

## 1. 배경 & 문제 정의

### 1.1 왜 지금인가

- **저출산 + 객단가 상승("골드키즈")**: 아이 수는 줄지만 1인당 양육 지출은 늘어, 한 아이에게 "제대로 된 것"을 사주려는 수요가 강하다. (출처 후보: 통계청 출생통계, 산업연구원 유아용품 시장 리포트 — `docs/데이터소스.md` 참조)
- **정보 과부하**: 초보 부모는 맘카페·블로그·유튜브·오픈마켓 리뷰를 직접 비교하며 의사결정 비용이 매우 높다.
- **개인화 부재**: 기존 커머스는 "잘 팔리는 순"으로만 보여줄 뿐, *우리 아기 성향과 발달 단계*에 맞춘 추천을 못 한다.

### 1.2 핵심 문제 (Pain Points)

| 사용자 | 핵심 페인 |
|--------|-----------|
| 초보 엄마·아빠 | "지금 시기에 뭐가 필요한지", "국민템이 우리 애한테도 맞는지" 모름 |
| 양육자 전반 | 검색·비교에 드는 시간, 잘못 산 용품(실패 구매)에 대한 후회 |
| 브랜드(업체) | 고관여·고전환 양육자에게 정밀하게 도달할 채널 부족 |

### 1.3 기존 대안의 한계

- **맘카페/블로그**: 신뢰도 높으나 광고성 글 혼재, 비체계적, 검색 피로.
- **오픈마켓(쿠팡/네이버쇼핑)**: 상품은 많지만 "성향·발달" 축의 개인화 추천이 없음.
- **육아 앱(베이비타임 등)**: 기록/정보 중심, *커머스 추천 + 수익화*와 결합 안 됨.

> **포지셔닝**: "육아 정보 앱"과 "오픈마켓" 사이의 빈 공간 — **성향·발달 기반 큐레이션 커머스**.

## 2. 타깃 사용자 (페르소나)

### 페르소나 A — 첫 아이 초보맘 "지원" (32세, 생후 4개월)
- 무엇을 사야 할지 매번 검색, 실패 구매 경험 있음.
- 니즈: "이 시기에 꼭 필요한 것"을 신뢰할 수 있게 정리해 주는 것.

### 페르소나 B — 둘째 양육 "현우" (38세, 36개월 + 8개월)
- 첫째 때 산 것 활용, 둘째 성향이 달라 고민.
- 니즈: 성향 차이에 따른 *맞춤* 추천, 시간 절약.

### 페르소나 C — 선물 구매 조부모/지인 "영자" (61세)
- 손주에게 좋은 걸 사주고 싶지만 트렌드를 모름.
- 니즈: "월령 + 무난한 국민템" 큐레이션, 쉬운 구매.

> MVP 1차 타깃은 **페르소나 A(첫 아이 0~24개월 양육자)** — 페인이 가장 크고 결정 빈도가 높다.

## 3. 핵심 가치 제안 (Value Proposition)

### 3.1 사용자(B2C) 측
- **시간 절약**: 검색·비교 대신 "우리 아기용" 추천 리스트.
- **실패 없는 구매**: 성향·월령 기반 + 설명 가능한 추천 이유.
- **발달 동행**: 월령이 바뀌면 다음 단계 용품/놀이를 선제 안내.

### 3.2 업체(B2B) 측
- **정밀 타깃팅**: "활동성 높은 18개월 아이 양육자"처럼 성향·월령 코호트 단위 도달.
- **고전환 채널**: 구매 의도가 명확한 양육자 풀.
- **데이터 인사이트**: 코호트별 수요/반응 리포트 제공.

## 4. 제품 정의 & 핵심 기능

### 4.1 사용자(B2C) 기능

1. **아기 프로필 & 성향 진단**
   - 입력: 생년월일(월령), 거주지, 형제 유무, 가정환경(주거형태 등), 예산대.
   - 성향 진단 설문(기질 기반 10~15문항) → 성향 프로필 생성. (상세: `docs/추천엔진_설계.md`)
2. **맞춤 추천 피드**
   - ① **국민템 추천** (월령·지역 코호트 인기) + ② **성향 맞춤 추천** + ③ **발달 놀이/교구 추천**.
   - 각 추천에 **"왜 추천하는가"** 설명(설명가능성) 표시 → 신뢰 확보.
3. **발달 타임라인 / 체크리스트**
   - 월령별 발달 과업 + 준비물 체크리스트(예: 이앓이 시기 → 치발기).
4. **위시리스트 · 비교 · 리뷰**
   - 저장, 스펙/가격 비교, 양육자 실사용 리뷰(성향 태그 포함 → "우리 애와 비슷한 성향" 필터).
5. **구매 연결**
   - 입점 브랜드 자사몰/제휴 링크로 연결(어필리에이트) 또는 향후 내부 결제.

### 4.2 업체(B2B) 기능

1. **입점/온보딩**: 브랜드·상품 등록, 카테고리/성향 태깅.
2. **광고 캠페인 관리**: 추천 상위 노출, 배너, 스폰서드 추천(명확한 "AD" 표기).
3. **성과 대시보드**: 노출·클릭·전환, 코호트별 반응 리포트.

### 4.3 신뢰 정책 (중요)
- 광고/스폰서 추천은 **항상 명시("AD"/"광고")**.
- 추천 알고리즘 점수와 광고 노출을 **분리**해 표기 → 추천 신뢰도 훼손 방지(플랫폼의 장기 해자).

## 5. 비즈니스 모델 (수익화)

### 5.1 수익원

| 수익원 | 설명 | 과금 방식 |
|--------|------|-----------|
| **입점/구독료 (B2B SaaS)** | 브랜드가 플랫폼 입점·노출·대시보드 사용 | 티어별 월/연 구독 |
| **광고** | 추천 상위 노출, 배너, 스폰서드 추천 | CPC/CPM/정액 |
| **제휴 커미션 (옵션)** | 구매 발생 시 어필리에이트 수수료 | 매출 연동 % |
| **데이터/인사이트 (확장)** | 코호트 수요 리포트(익명·집계) | 별도 상품 |

### 5.2 입점 티어 (예시)

| 티어 | 월 구독료(예시) | 제공 |
|------|------------------|------|
| Basic | 무료/저가 | 상품 등록, 기본 노출, 기본 통계 |
| Pro | 중가 | 추천 가중치 입찰, 캠페인, 상세 리포트 |
| Premium | 고가 | 코호트 타깃 광고, 전담 지원, API |

> 단가는 시장 검증(영업/파일럿) 후 확정. 위 표는 구조 설계용 예시.

### 5.3 수익 흐름 (텍스트 다이어그램)

```
                ┌─────────────────────────────────────┐
   (무료)       │            플랫폼 (baby_predict)       │
 양육자 B2C ───▶│  성향 진단 · 추천 · 발달 콘텐츠          │◀─── 브랜드 B2B
   사용 ◀───────│                                        │   입점료/구독료 💰
   추천/구매    │  추천 엔진 + 광고 엔진 + 데이터          │   광고비 💰
        │       └─────────────────────────────────────┘
        │                       │
        ▼                       ▼
   구매 전환 ───▶ 어필리에이트 커미션 💰        코호트 인사이트 리포트 💰
```

> **플라이휠**: 양육자 데이터↑ → 추천 정확도↑ → 사용자/전환↑ → 브랜드 유입↑ → 콘텐츠/혜택↑ → 양육자↑.

### 5.4 단위 경제(예시 시뮬레이션, 가정치)
- 가정: 입점 브랜드 100개 × 평균 구독 ARPA 30만 원/월 = **월 3,000만 원** (구독).
- 광고: MAU 5만, 추천 클릭 광고 단가/전환 기반 별도 추정.
- *실제 수치는 파일럿 데이터로 대체 — 본 문서는 구조 검증용.*

## 6. 추천/예측 엔진 개요

> 상세 설계는 **`docs/추천엔진_설계.md`** 참조.

- **입력 피처**: 아기 성향(기질 벡터), 월령, 거주지(기후/주거), 형제 유무, 예산, (축적 후) 행동·구매 로그.
- **출력**: 카테고리별 추천 아이템 + 적합도 점수 + **추천 이유**.
- **단계별 고도화**:
  1. **규칙기반 + 지식 매핑**(콜드스타트): 월령→발달 과업→용품 카테고리 매핑.
  2. **인기도/국민템 랭킹**: 지역·연령 코호트별.
  3. **협업 필터링**: 행동·구매 로그 축적 후.
  4. **하이브리드 + ML 랭킹**: 콘텐츠·CF·맥락 통합.

## 7. 데이터 전략

### 7.1 자체 수집 (1st-party)
- 프로필(성향 진단), 행동 로그(클릭/저장/검색), 구매/전환, 리뷰.
- → 시간이 지날수록 정확도가 복리로 좋아지는 **핵심 자산**.

### 7.2 외부 데이터 후보 (근거 보강용)
- **Naver 데이터랩 쇼핑**: 연령/성별별 육아용품 트렌드·인기 키워드 → "국민템" 랭킹 근거.
- **kidshub(공공)**: 영유아 성장 분석, 연령별 인기 도서, 예방접종 일정 → 발달 단계 콘텐츠.
- **공공데이터/통계청**: 출생·인구, 지역 코호트.
- 상세: `docs/데이터소스.md`.

### 7.3 콜드스타트 해결
- 신규 사용자/신규 상품 모두 **규칙기반 + 인기도**로 첫 추천 → 로그 쌓이면 개인화로 전환.

## 8. 기술 아키텍처 (하이레벨)

```
[모바일/웹 클라이언트]
        │ (REST/GraphQL)
        ▼
[API 게이트웨이 / 백엔드]
   ├─ 사용자/프로필 서비스
   ├─ 추천 서비스 ──▶ [추천 엔진]  ◀── 피처 스토어
   ├─ 광고/입찰 서비스
   ├─ 카탈로그(상품) 서비스
   └─ 업체 포털(B2B) 서비스
        │
        ▼
[데이터 레이어]  운영 DB(PostgreSQL) · 이벤트 로그 · 데이터 웨어하우스 · ML 파이프라인
```

### 권장 스택 후보 (선택 근거 포함)
- **프런트**: 웹 Next.js + 모바일(React Native/Flutter) — 코드 공유·빠른 출시.
- **백엔드**: Python(FastAPI) 또는 Node(NestJS) — Python은 ML 통합 유리.
- **DB**: PostgreSQL(운영) + 이벤트 로그(BigQuery/ClickHouse 등).
- **추천**: 초기 규칙엔진 → scikit-learn/LightFM → 추후 딥러닝.
- *스택은 팀 역량에 맞춰 확정. ML 비중 큰 만큼 Python 백엔드 권장.*

## 9. 로드맵 / MVP

| 단계 | 기간(목표) | 핵심 내용 | 추천 방식 | 수익화 |
|------|-----------|-----------|-----------|--------|
| **Phase 0 — MVP** | 0~3개월 | 성향 진단 + 발달 타임라인 + 추천 피드(웹) | 규칙기반 + 국민템 인기 랭킹 | 수동 입점(영업), 제휴 링크 |
| **Phase 1** | 3~6개월 | 앱 출시, 리뷰/위시리스트, 업체 셀프서브 대시보드 | + 협업 필터링 | 구독 티어 + 기본 광고 |
| **Phase 2** | 6~12개월 | ML 예측 모델, 광고 입찰 플랫폼, 인사이트 리포트 | 하이브리드 ML 랭킹 | 광고 플랫폼 고도화 |

> **MVP 성공 기준(가설 검증)**: 진단 완료율, 추천 클릭률, 제휴 전환 → "성향 기반 추천이 인기순보다 전환이 높은가?"

## 10. 사용자 플로우 (화면 흐름)

### B2C — 양육자
```
온보딩(가입) → 아기 프로필 입력 → 성향 진단 설문
   → [추천 피드: 국민템 + 성향맞춤 + 발달놀이]
   → 상품 상세/비교/리뷰 → 위시리스트/구매(제휴링크)
   → (월령 변화 알림) → 다음 단계 추천 재방문
```

### B2B — 브랜드
```
업체 가입 → 브랜드/상품 등록 + 성향·카테고리 태깅
   → 캠페인 생성(타깃 코호트 선택) → 노출/입찰
   → 성과 대시보드(노출·클릭·전환·코호트 반응)
```

|--------|------|------|
| **개인정보** | 영유아·양육자 민감정보(생년월일·지역) | 개인정보보호법 준수, 최소 수집·동의, 만 14세 미만 법정대리인 동의, 가명·암호화 |
| **표현 규제** | "발달/건강" 표현이 의료·과장광고로 오인 | 의료 효능 표현 금지, "발달 단계 참고" 톤, 표시광고법 준수 |
| **광고 표시** | 스폰서 추천의 기만 우려 | "광고/AD" 명확 표기, 추천점수와 분리 |
| **추천 편향/신뢰** | 광고가 추천을 잠식 | 알고리즘·광고 슬롯 분리, 품질 가드레일 |
| **입점사 의존** | 초기 브랜드 부족(닭-달걀) | 제휴 링크로 카탈로그 우선 확보 후 입점 영업 |
| **콜드스타트** | 데이터 없는 초기 추천 품질 | 규칙기반+공공/트렌드 데이터로 보완 |

---

## 13. 경쟁 우위 / 해자 (Moat)

1. **성향–발달–구매 데이터 축적**: 시간이 갈수록 추천 정확도가 복리로 향상 → 후발주자가 따라오기 어려움.
2. **설명 가능한 추천 + 신뢰 정책**: "왜 추천하는지" + 광고 분리 → 양육자 신뢰가 곧 리텐션.
3. **양면 네트워크 효과**: 양육자↑ → 브랜드↑ → 혜택/콘텐츠↑ → 양육자↑.
4. **월령 기반 생애주기 락인**: 아이가 자라며 계속 다음 단계 추천을 받는 구조.

## 부록 A. 용어
- **국민템**: 해당 월령 코호트에서 압도적으로 많이 구매·추천되는 표준 인기 용품.
- **성향(기질)**: 아기의 활동성·규칙성·적응성·반응강도 등 행동 경향(상세는 추천엔진 문서).
- **코호트**: 월령·지역·성향 등으로 묶인 사용자 집단.
