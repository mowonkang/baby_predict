# 추천/예측 엔진 상세 설계

> 상위 문서: [`docs/기획안.md`](기획안.md)
> 본 문서는 "아기 성향 + 월령 + 환경 → 발달 맞춤 용품·놀이 추천" 엔진의 설계를 다룬다.

---

## 1. 목표

입력된 아기 정보로부터 **(a) 국민템, (b) 성향 맞춤 아이템, (c) 발달 단계 놀이/교구**를 추천하고, 각 추천에 **설명(추천 이유)**을 붙인다. 데이터가 없는 초기(콜드스타트)에도 합리적 추천이 가능해야 한다.

---

## 2. 입력 피처 (Feature Set)

| 피처 그룹 | 항목 | 설명 |
|-----------|------|------|
| **아기 성향(기질)** | 활동성, 규칙성, 적응성, 반응 강도, 기분 경향 | 진단 설문으로 산출한 성향 벡터 |
| **발달 단계** | 월령(생년월일 기반) | 가장 강력한 단일 신호 |
| **환경** | 거주지(기후·지역), 주거형태, 형제 유무 | 예: 추운 지역 → 보온용품 가중 |
| **경제** | 예산대, 가격 민감도 | 추천 가격 범위 조정 |
| **행동(축적 후)** | 클릭/저장/검색/구매 로그 | 협업필터링·개인화 |

### 2.1 성향(기질) 모델
- 검증된 아동 기질 이론(예: Thomas & Chess의 9개 기질 차원 → 순한 아이 / 까다로운 아이 / 더딘 아이 분류)을 **참고**해 설문 설계.
- 출력: 정규화된 **성향 벡터** `temperament = {activity, regularity, adaptability, intensity, mood}` (각 0~1).
- 주의: 의학적 진단이 아닌 **추천용 선호 프로파일**로 한정 (규제·신뢰 이슈, 기획안 §12 참조).

### 2.2 성향 진단 설문 (예시 설계)
- 10~15문항, 상황형 문항(예: "새로운 환경에서 우리 아기는?" → 보기 5점 척도).
- 각 문항이 특정 기질 차원에 가중치로 매핑 → 합산·정규화 → 성향 벡터.
- 월령에 따라 문항 셋 분기(영아/유아).

---

## 3. 출력 (Output)

```json
{
  "recommendations": [
    {
      "category": "수면",
      "item_id": "SKU123",
      "score": 0.92,
      "type": "national_pick | temperament_match | developmental_play",
      "reasons": [
        "생후 6개월 또래에서 가장 많이 선택된 국민템",
        "활동성이 높은 아기에게 적합한 안전 설계"
      ]
    }
  ]
}
```

- **score**: 적합도(0~1), 랭킹 정렬용.
- **reasons**: 설명가능성(신뢰 핵심) — 규칙/모델 근거를 사람 언어로 변환.
- 광고 슬롯은 별도 필드로 분리(추천 score와 섞지 않음).

---

## 4. 단계별 모델 전략 (점진적 고도화)

### Stage 1 — 규칙기반 + 지식 매핑 (콜드스타트 / MVP)
- **월령 → 발달 과업 → 용품/놀이 카테고리** 매핑 테이블(아래 §5)로 후보 생성.
- 성향·환경 규칙으로 가중치 조정(예: 활동성↑ → 안전·대근육 교구 가중).
- 장점: 데이터 0에서도 동작, 설명 쉬움. 단점: 개인화 한계.

### Stage 2 — 인기도 / 국민템 랭킹
- **코호트(월령×지역×성향)별 인기 아이템** 집계로 "국민템" 산출.
- 데이터 소스: 자체 구매/저장 로그 + 외부 트렌드(Naver 데이터랩 쇼핑 등, `docs/데이터소스.md`).

### Stage 3 — 협업 필터링 (CF)
- 행동·구매 로그 축적 후 user–item 행렬 기반 CF(예: implicit ALS / LightFM).
- "비슷한 성향·월령 양육자가 함께 산 아이템" 추천.

### Stage 4 — 하이브리드 + ML 랭킹
- 콘텐츠 기반(아이템 속성) + CF + 맥락(성향·환경) 피처를 통합한 **러닝-투-랭크** 모델.
- 최종 점수 = f(적합도, 인기도, 개인화, 신선도) — 가중치 학습/튜닝.

---

## 5. 발달 단계 매핑 테이블 (예시)

> 실제 운영 테이블은 전문가 검수 + 공공 데이터(kidshub 성장/도서, 예방접종 일정 등) 기반으로 확장.

| 월령 | 대표 발달 과업 | 추천 용품 카테고리 | 추천 놀이/교구 |
|------|----------------|--------------------|-----------------|
| 0~3개월 | 수면·수유 안정, 시각 자극 | 수유용품, 속싸개, 모빌 | 흑백 모빌, 초점책 |
| 4~6개월 | 뒤집기, 이앓이 시작 | 치발기, 보행 전 매트 | 딸랑이, 촉감책 |
| 7~9개월 | 앉기·기기, 손가락 잡기 | 이유식 용품, 안전용품 | 쌓기 컵, 소리 장난감 |
| 10~12개월 | 잡고 서기, 첫걸음 준비 | 보행기/푸시워커, 신발 | 모양 맞추기, 공놀이 |
| 13~18개월 | 걷기, 단어 시작 | 외출용품, 식기 | 끌기 장난감, 그림책 |
| 19~24개월 | 뛰기, 두 단어 문장 | 대근육 교구 | 블록, 역할놀이 |
| 24개월+ | 사회성·상상놀이 | 학습·창의 교구 | 퍼즐, 미술놀이 |

- **성향 보정 예시**: 같은 12개월이라도 *활동성↑* → 대근육 교구 가중, *반응강도↑(예민)* → 자극 적은 수면·진정 용품 가중.

---

## 6. 추천 파이프라인 (런타임)

```
입력(프로필+성향) 
   → [후보 생성]  규칙 매핑 + 인기 코호트 + (CF 후보)
   → [필터링]     예산/안전/중복/이미 보유 제외
   → [랭킹]       적합도·인기·개인화·신선도 가중 점수
   → [설명 생성]  점수 기여 요인 → reasons 텍스트
   → [광고 병합]  별도 슬롯에 스폰서 노출(AD 표기)
   → 추천 피드 반환
```

---

## 7. 평가 (Evaluation)

### 7.1 오프라인
- **정확도**: precision@k, recall@k, NDCG.
- **다양성/신선도**: 카테고리 커버리지, 중복도.
- **콜드스타트 품질**: 신규 사용자 첫 추천 만족도(프록시 지표).

### 7.2 온라인
- **A/B 테스트**: 규칙기반 vs CF vs 하이브리드 → 추천 CTR, 위시리스트 추가, **전환율** 비교.
- 핵심 가설: *"성향 기반 추천이 단순 인기순보다 전환이 높은가?"*

### 7.3 가드레일
- 안전(리콜/연령 부적합 제외), 광고-추천 분리, 편향 모니터링.

---

## 8. 콜드스타트 요약

| 대상 | 데이터 없음 시 |
|------|----------------|
| 신규 사용자 | 월령 규칙 + 코호트 인기(국민템) 우선 |
| 신규 상품 | 콘텐츠(속성) 기반으로 적합 코호트에 노출 |
| 신규 지역 | 전국/유사지역 인기로 대체 |

---

## 9. 향후 확장
- 양육자 피드백 루프(추천 "도움됨/아니요")로 온라인 학습.
- 멀티모달(상품 이미지·리뷰 텍스트) 피처.
- 재구매·소진 주기 예측(소모품 자동 추천 타이밍).
